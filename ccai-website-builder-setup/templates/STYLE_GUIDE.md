## Components

### Buttons
- Primary CTA: shadcn `<Button variant="default">`, uses primary color
- Secondary CTA: `<Button variant="outline">`
- Ghost: `<Button variant="ghost">` for nav links
- All buttons have hover states from shadcn defaults

### Cards
- Use shadcn `<Card>` for any content block with a border
- Default radius: `rounded-lg`
- Shadow: `shadow-sm` (subtle, no heavy shadows)

### Forms
- Use shadcn `<Input>` and `<Form>` components
- Labels always visible (no placeholder-as-label)
- Validation messages below the input

## Animation

Framer Motion for any animation beyond CSS hovers.

### Default page entry
- `initial={{ opacity: 0, y: 20 }}`
- `animate={{ opacity: 1, y: 0 }}`
- `transition={{ duration: 0.4, ease: "easeOut" }}`

### Scroll reveals
Use `whileInView` with `viewport={{ once: true }}` for one-time reveals.

### What NOT to animate
- Don't animate the text itself (just the container)
- Don't bounce, feels dated
- No flips, spins, or zoom-ins on hero text
- Animation should be invisible to most users, felt, not seen

## Accessibility defaults

- All interactive elements must have visible focus states (use Tailwind `focus-visible:` utilities)
- Color contrast ≥ WCAG AA (4.5:1 for body text, 3:1 for large text)
- All images need `alt` attributes (decorative images use `alt=""`)
- Form labels always associated with inputs (`htmlFor` / `id`)
- No motion-required animations (respect `prefers-reduced-motion`)
